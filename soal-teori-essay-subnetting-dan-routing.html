1. 1. Subnetting 2. VLSM 3. Routing 4. Application layer


Jawaban:

1. teknik yang digunakan untuk membagi sebuah jaringan menjadi beberapa bagian kecil yang disebut subnet

2. VLSM (Variable Length Subnet Mask) adalah teknik yang digunakan untuk membagi jaringan menjadi beberapa subnet dengan menggunakan subnet mask yang berbeda-beda sesuai dengan kebutuhan

3. Routing adalah proses menentukan jalur yang akan ditempuh oleh paket data agar sampai ke tujuannya

4. Application layer adalah lapisan teratas dari model arsitektur jaringan yang disebut model OSI (Open Systems Interconnection).

Penjelasan:

Penjelasan jawaban no 1

Subnetting adalah teknik yang digunakan untuk membagi sebuah jaringan menjadi beberapa bagian kecil yang disebut subnet. Tujuan dari subnetting adalah untuk mengelola jaringan dengan lebih efisien dan meningkatkan keamanan jaringan. Dengan menggunakan subnetting, administrator jaringan dapat mengontrol akses ke jaringan dan mengalokasikan sumber daya jaringan secara lebih efektif.

Subnetting menggunakan alamat IP (Internet Protocol) yang terdiri dari 32 bit. Sebuah alamat IP dapat dibagi menjadi dua bagian, yaitu bagian yang menunjukkan identitas jaringan (network ID) dan bagian yang menunjukkan identitas host (host ID). Dengan menggunakan subnetting, administrator jaringan dapat menentukan berapa banyak bit yang akan digunakan untuk network ID dan berapa banyak bit yang akan digunakan untuk host ID.

Penjelasan jawaban no 2

VLSM adalah singkatan dari Variable Length Subnet Mask. VLSM merupakan sebuah teknik yang digunakan dalam jaringan komputer untuk membagi sebuah jaringan ke dalam subnet yang lebih kecil dengan menggunakan subnet mask yang berbeda-beda sesuai dengan kebutuhan. VLSM memungkinkan penggunaan alokasi alamat IP yang lebih efisien dan menghemat alokasi alamat IP dalam suatu jaringan.

Penjelasan jawaban no 3

Routing adalah proses menentukan jalur yang akan ditempuh oleh paket data agar sampai ke tujuannya. Routing biasanya dilakukan oleh perangkat jaringan seperti router, yang memiliki tabel routing yang menyimpan informasi tentang jalur terbaik untuk mengirim paket data ke tujuannya.

penjelasan jawaban no 4

Application layer adalah lapisan teratas dari model arsitektur jaringan yang disebut model OSI (Open Systems Interconnection). Lapisan aplikasi bertanggung jawab untuk menyediakan layanan dan antarmuka yang dibutuhkan oleh aplikasi yang berjalan pada sistem komputer. Lapisan ini memungkinkan aplikasi untuk berkomunikasi dengan lapisan lain dari model OSI melalui protokol yang sesuai. Contohnya, protokol HTTP (Hypertext Transfer Protocol) digunakan untuk mentransfer data di World Wide Web, sedangkan protokol FTP (File Transfer Protocol) digunakan untuk mentransfer file antar sistem

1. soal: Host A is connected to the LAN, but it cannot connect to the Internet. The host configuration is shown in the exhibit. What are the two problems with this configuration?

Jawab: CIDR /27 artinya netmask yang digunakan adalah 255.255.255.224. Dari sini kita tahu bahwa isian netmask di host adalah berbeda, jadi salah setting di netmask. Yang kedua blok subnet = 256-224 = 32, jadi subnetnya adalah kelipatan 32 (0, 32, 64, 86, 128, …, 224). Artinya di bawah Router 1, masuk di subnet 198.18.166.32. Alamat gateway sudah benar, karena biasa digunakan alamat host pertama. Hanya alamat IP hostnya salah karena 198.18.166.65 masuk di alamat subnet 198.18.166.64 dan bukan 198.18.166.32.

2 . Soal: What is the subnetwork number of a host with an IP address of 172.16.66.0/21?

Jawab: CIDR /21 berarti 255.255.248.0. Blok subnet = 256- 248 = 8, netmasknya adalah kelipatan 8 (0, 8, 16, 24, 32, 40, 48, …, 248) dan karena ini adalah alamat IP kelas B, blok subnet kita “goyang” di oktet ke 3. Tidak perlu kita list semuanya, kita hanya perlu cari kelipatan 8 yang paling dekat dengan 66 (sesuai dengan soal), yaitu 64 dan 72. Jadi susunan alamat IP khusus untuk subnet 172.16.64.0 dan 172.16.72.0 adalah seperti di bawah. Jadi pertanyaan bisa dijawab bahwa 172.16.66.0 itu masuk di subnet 172.16.64.0

27. jelaskan perbedaan static routing dan dynamic routing


Perbedaan static routing dan dynamic routing adalah :

Segi Komplektisitas

Static Routing akan semakin kompleks dalam konfigurasi seiring dengan besarnya jaringan

Dynamic Routing dalam tingkat kekompleksan untuk konfigurasinya tidak bergantung pada seberapa besar jaringan yang dibuat

Segi Penyesuaian topologi

Static routing akan memerlukan campur tangan administrator jaringan saat mengubah suatu topologi sedangkan dynamic routing akan menyesuaikan secara otomatis.

Dalam hal ini berimbas pada fakta bahwa static routing lebih cocok untuk topologi yang simple dan dynamic routing lebih fleksibel untuk menyesuaikan apakah topologi yang digunakan kompleks atau tidak.

Segi Keamanan

Static routing lebih aman daripada dynamic routing

Segi Penggunaan Resources

Static routing tidak memerlukan tambahan perangkat namun, dynamic routing membutuhkan cpu, memori dan bandwidth

Segi Alur

Static Routing memilik rute yang akan selalu sama, akan tetapi dynamic routing rutenya bergantung bagaimana kondisi dan bentuk topologinya

Pembahasan

Routing adalah suatu proses menyalurkan data - data antar komputer dalam suatu jaringan maupun antar jaringan. Biasanya data yang disalurkan berbentuk paket data. Sebenarnya, ditinjau dari jenis konfigurasinya routing dibagi menjadi tiga yaitu static routingg, dynamic routing dan minimal routing. Untuk minimal routing, konfigurasi sangat sederhana dan hanya digunakan lokal. Sedangkan berdasarkan model pengiriman dibagi menjadi routing langsung dan tidak langsung.

34. JELASKAN PERBEDAAN ANTARA ROUTING STATIC DAN ROUTING DINAMIS


Perbedaan yang berada di antara routing static dan routing dinamis adalah:

Routing StatikMemiliki fungsi yang terdapat pada protocol IPRouter tidak akan dapat melakukan pembagian daripada informasi routingRouting table akan dilakukan pembuatan dan juga dihapuskan secara manualTidak akan menggunakan berbagai macam bentuk routing protocolMicrosoft akan memberikan dukungan kepada multihomed system yang dimana akan seperti sebuah router.Routing DinamikMemiliki fungsi pada bagian inter-routing protocolRouter akan melakukan pembagian daripada informasi routing yang dilakukan secara otomatisRouting table yang dibuat akan dilakukan penghapusan secara otomatisAkan terdapat sebuah routing protokol seperti RIP hingga OSPFMicrosoft akan memberikan dukungan kepada RIP terhadap IP dan IPX maupun SPXPembahasan

Static routing adalah sebuah bentuk router yang dimana akan memiliki berbagai macam bentuk tabel routing statik yang akan berada pada bagian setting yang dilakukan secara manual kepada para administrator jaringan. Selanjutnya routing static daripada pengaturan routing paling sederhana yang dimana akan dapat digunakan pada sebuah jaringan komputer.
